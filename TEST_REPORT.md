### 3. 웹 검색 통합 테스트 ⚠️

부분적 성공 (2/3, 1 정보)

#### 테스트 항목

- ❌ **"웹 검색" 선택 시 DuckDuckGo 검색 작동**
  - API: `POST /api/search/unified` (sources: ["web"])
  - 문제: 웹 검색 결과가 반환되지 않음
  - 원인 분석:
    - DuckDuckGo API가 Docker 컨테이너 환경에서 제한될 수 있음
    - 네트워크 제한 또는 rate limiting 가능성
    - API 자체는 정상 작동 (200 OK, 단지 결과가 0개)
  
- ℹ️ **통합 검색 결과에 웹 검색 결과 포함**
  - 결과: 웹 검색 결과가 없어 통합 결과에 포함되지 않음
  - 참고: 문서 및 지식베이스 검색은 정상 작동
  
- ✅ **에러 처리 (잘못된 소스)**
  - 결과: 잘못된 소스 타입이 적절히 처리됨
  - 시스템이 안정적으로 에러 처리

#### 권장 사항

웹 검색은 **선택적 기능**으로 처리:
- 사용자에게 웹 검색이 현재 제한될 수 있음을 안내
- 문서 + 지식베이스 검색만으로도 충분히 작동
- 프로덕션 환경에서 외부 API 키 사용 고려 (Serper API, Brave Search API 등)

## 발견된 버그 및 수정 사항

### 수정 완료

1. **가중치 기반 재정렬 문제**
   - 문제: 음수 relevance_score로 인한 정렬 오류
   - 해결: `unified_search.py`의 `_merge_results()` 메서드에 점수 정규화 로직 추가
   - 변경 사항:
     ```python
     # 점수 정규화: 코사인 유사도(-1 ~ 1)를 0 ~ 1 범위로 변환
     if original_score < 0:
         normalized_score = (original_score + 1) / 2
     else:
         normalized_score = original_score
     ```

### 알려진 제한사항

1. **웹 검색 결과 없음**
   - 현상: DuckDuckGo 검색 API가 결과를 반환하지 않음
   - 가능한 원인:
     - Docker 컨테이너 네트워크 제한
     - DuckDuckGo API rate limiting
     - 지역 제한
   - 해결 방안:
     - 프로덕션 환경에서 유료 검색 API 사용 (Serper, Brave Search)
     - 웹 검색을 선택적 기능으로 유지
     - 문서 + 지식베이스만으로도 충분히 작동
