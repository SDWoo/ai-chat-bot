# 스트리밍 기능 테스트 가이드

## 구현 완료 내역

### ✅ 백엔드 구현
1. **SSE (Server-Sent Events) 엔드포인트** (`/api/chat/stream`)
   - Content-Type: `text/event-stream`
   - 실시간 청크 단위 스트리밍
   - 메타데이터, 컨텐츠, 완료 신호 전송
   - 스트리밍 완료 후 DB에 메시지 자동 저장

2. **에러 처리**
   - Try-catch 블록으로 예외 처리
   - 에러 발생 시 클라이언트에 에러 타입 전송
   - 로깅을 통한 디버깅 지원

### ✅ 프론트엔드 구현
1. **스트리밍 API 클라이언트**
   - AsyncGenerator를 사용한 스트리밍 처리
   - SSE 형식 파싱 (`data: ` 접두사)
   - 자동 재시도 로직 (최대 3회)
   - 타임아웃 처리 (120초)

2. **UI 스트리밍 상태 관리**
   - 실시간 메시지 업데이트
   - 타이핑 애니메이션 (파란색 커서)
   - 처리 상태 표시 (문서 검색 중, 답변 생성 중)
   - 재시도 시 시각적 피드백 (오렌지색 텍스트)

3. **에러 처리 및 재시도**
   - 네트워크 에러 시 최대 3회 재시도
   - 재시도 간격: 1초, 2초, 3초 (지수적 증가)
   - 4xx 에러는 재시도하지 않음
   - 타임아웃 시 재시도

## 테스트 체크리스트

### 1. 기본 스트리밍 테스트
- [ ] 백엔드 서버 시작
- [ ] 프론트엔드 서버 시작
- [ ] 문서 업로드
- [ ] 간단한 질문 입력 (예: "이 문서의 주제는?")
- [ ] 답변이 한 글자씩 실시간으로 나타나는지 확인
- [ ] 타이핑 애니메이션 (파란색 커서) 표시 확인
- [ ] "문서 검색 중..." → "답변 생성 중..." 상태 전환 확인
- [ ] 답변 완료 후 참고 문서 표시 확인

### 2. 긴 답변 스트리밍 테스트
- [ ] 복잡한 질문 입력 (예: "이 문서의 내용을 자세히 설명해주세요")
- [ ] 답변이 끊김 없이 스트리밍되는지 확인
- [ ] 마크다운 렌더링이 실시간으로 업데이트되는지 확인
- [ ] 스크롤이 자동으로 하단으로 이동하는지 확인

### 3. 연속 메시지 테스트
- [ ] 첫 번째 질문 입력 후 답변 완료 대기
- [ ] 두 번째 질문 입력 (답변이 스트리밍되는 동안 입력 불가 확인)
- [ ] 두 번째 답변이 정상적으로 스트리밍되는지 확인
- [ ] 대화 히스토리가 올바르게 유지되는지 확인

### 4. 에러 처리 테스트
#### 4.1 네트워크 중단 시뮬레이션
- [ ] 개발자 도구 → Network 탭 열기
- [ ] 질문 입력 후 즉시 Network를 Offline으로 변경
- [ ] "연결 재시도 중... (1/3)" 메시지 표시 확인
- [ ] Network를 Online으로 복구
- [ ] 재시도 후 정상 스트리밍 확인

#### 4.2 타임아웃 테스트
- [ ] 백엔드 서버 중지
- [ ] 질문 입력
- [ ] 재시도 메시지 표시 확인 (3회)
- [ ] 최종 에러 메시지 표시 확인

#### 4.3 서버 에러 테스트
- [ ] 백엔드를 다시 시작하되 OpenAI API 키를 잘못 설정
- [ ] 질문 입력
- [ ] 에러 메시지 표시 확인

### 5. 동시성 테스트
- [ ] 브라우저 탭 2개 열기
- [ ] 두 탭에서 동시에 다른 질문 입력
- [ ] 각 탭에서 독립적으로 스트리밍되는지 확인
- [ ] 대화가 섞이지 않는지 확인

### 6. 성능 테스트
- [ ] 크롬 개발자 도구 → Network 탭에서 스트리밍 요청 확인
- [ ] Time to First Byte (TTFB) 확인
- [ ] 청크가 실시간으로 도착하는지 확인
- [ ] 메모리 사용량 확인 (개발자 도구 → Performance Monitor)

## 테스트 시작 방법

### 1. 환경 설정
```bash
# .env 파일에서 OpenAI API 키 확인
OPENAI_API_KEY=your-api-key-here
OPENAI_MODEL=gpt-4o-mini
```

### 2. 백엔드 서버 시작
```bash
cd backend
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3. 프론트엔드 서버 시작
```bash
cd frontend
npm run dev
```

### 4. 브라우저에서 접속
```
http://localhost:5173
```

## 예상 성능 개선 효과

### Before (비스트리밍)
- 평균 응답 시간: **10초**
- 사용자가 느끼는 대기 시간: **10초** (화면 변화 없음)
- 사용자 경험: ⭐⭐ (답답함)

### After (스트리밍)
- 평균 응답 시간: **10초** (동일)
- 첫 번째 단어 표시: **1-2초**
- 사용자가 느끼는 대기 시간: **1-2초**
- 사용자 경험: ⭐⭐⭐⭐⭐ (즉각적인 피드백)

### 주요 개선 사항
1. **체감 대기 시간 80% 감소** (10초 → 1-2초)
2. **실시간 피드백** - 답변이 생성되는 과정을 볼 수 있음
3. **더 나은 사용자 경험** - ChatGPT와 유사한 UX
4. **네트워크 안정성** - 재시도 로직으로 일시적 장애 대응
5. **진행 상태 표시** - 현재 어떤 작업 중인지 명확히 표시

1. Chrome DevTools → Network 탭
2. "stream" 필터링
3. 요청 선택 → Response 탭
4. SSE 이벤트가 실시간으로 도착하는지 확인

## 알려진 제한사항

1. **브라우저 호환성**: EventSource는 IE에서 지원되지 않음 (하지만 fetch API 사용으로 해결)
2. **타임아웃**: 2분 이상 걸리는 요청은 타임아웃됨
3. **재시도 횟수**: 최대 3회로 제한
4. **동시 요청**: 한 번에 하나의 스트리밍 요청만 가능
