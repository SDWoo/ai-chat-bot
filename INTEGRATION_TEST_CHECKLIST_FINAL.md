  1. 백엔드 컨테이너 시작
  2. 로그 확인
  3. SyntaxError 발생
- **예상 동작**: 백엔드가 정상적으로 시작되어야 함
- **실제 동작**: Python 문법 오류로 인해 백엔드 시작 실패
- **관련 파일**: `backend/app/api/routes/chat.py` (라인 31, 45, 66)
- **해결**: f-string 내 여러 줄 딕셔너리를 별도 변수로 분리

**수정 내용**:
```python
# 수정 전 (오류)
yield f"data: {json.dumps({
    'type': 'metadata',
    'conversation_id': conversation_id,
})}\n\n"

# 수정 후 (정상)
metadata = {
    'type': 'metadata',
    'conversation_id': conversation_id,
}
yield f"data: {json.dumps(metadata)}\n\n"
```
