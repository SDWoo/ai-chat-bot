# 프론트엔드 UI 수동 테스트 체크리스트

이 문서는 프론트엔드 UI의 수동 테스트를 위한 체크리스트입니다.

## 사전 준비

- [ ] 백엔드 서버 실행 중 (`docker-compose up`)
- [ ] 프론트엔드 개발 서버 실행 중 (`npm run dev`)
- [ ] 브라우저에서 http://localhost:5173 접속

---

## 1. 지식 관리 페이지 테스트

### 1.1 페이지 접근
- [ ] 상단 메뉴에서 "지식 관리" 클릭
- [ ] 페이지가 정상적으로 로드됨
- [ ] 기존 지식 항목 목록이 표시됨

### 1.2 새 지식 추가
- [ ] "새 지식 추가" 버튼 클릭
- [ ] 모달 창이 나타남
- [ ] 제목 입력: "테스트 지식 항목"
- [ ] 내용 입력: "이것은 UI 테스트를 위한 항목입니다."
- [ ] 카테고리 선택: "사용법"
- [ ] 태그 입력 후 Enter: "UI테스트", "프론트엔드"
- [ ] 작성자 입력: "테스터"
- [ ] "생성" 버튼 클릭
- [ ] 모달이 닫히고 목록에 새 항목이 표시됨

### 1.3 지식 편집
- [ ] 방금 생성한 항목에서 편집 버튼(연필 아이콘) 클릭
- [ ] 편집 모달이 열리고 기존 데이터가 로드됨
- [ ] 제목을 "테스트 지식 항목 (수정됨)"으로 변경
- [ ] 태그 추가: "수정완료"
- [ ] "저장" 버튼 클릭
- [ ] 목록에 수정된 내용이 반영됨

### 1.4 필터링 및 검색
- [ ] 왼쪽 사이드바에서 카테고리 필터 선택
- [ ] 해당 카테고리 항목만 표시됨
- [ ] 검색 바에 키워드 입력
- [ ] 검색 결과가 실시간으로 필터링됨

### 1.5 지식 삭제
- [ ] 테스트 항목에서 삭제 버튼(휴지통 아이콘) 클릭
- [ ] 확인 대화상자 표시됨
- [ ] "확인" 클릭
- [ ] 항목이 목록에서 제거됨

---

## 2. 대화 학습 기능 테스트

### 2.1 대화 시작
- [ ] 상단 메뉴에서 "채팅" 클릭
- [ ] 채팅 페이지가 로드됨

### 2.2 검색 소스 선택
- [ ] 하단에 검색 소스 체크박스가 표시됨
- [ ] "📄 문서" 체크박스 확인
- [ ] "💡 지식베이스" 체크박스 확인
- [ ] "🌐 웹 검색" 체크박스 확인
- [ ] 각 체크박스를 클릭하여 활성화/비활성화 가능

### 2.3 대화 진행
- [ ] 메시지 입력: "FastAPI의 의존성 주입에 대해 설명해주세요"
- [ ] "전송" 버튼 클릭
- [ ] "문서 검색 중..." 상태 표시됨
- [ ] "답변 생성 중..." 상태 표시됨
- [ ] AI 답변이 스트리밍으로 표시됨
- [ ] 참고 문서 목록이 표시됨

### 2.4 피드백 제공
- [ ] AI 답변 하단에 👍👎 버튼이 표시됨
- [ ] 👍 버튼 클릭
- [ ] 버튼 색상이 변경됨 (피드백 저장됨)

### 2.5 지식으로 저장
- [ ] 긍정 피드백 후 "지식으로 저장" 버튼이 표시됨
- [ ] 버튼에 📑 아이콘과 텍스트가 표시됨
- [ ] "지식으로 저장" 버튼 클릭
- [ ] "지식 추출 중..." 로딩 표시
- [ ] 성공 메시지 alert 표시 (제목, 카테고리, 상태 정보 포함)
- [ ] alert 닫기

### 2.6 추출된 지식 확인
- [ ] 상단 메뉴에서 "지식 관리" 클릭
- [ ] 왼쪽 사이드바에서 "전체" 또는 해당 카테고리 선택
- [ ] 추출된 지식 항목이 목록에 표시됨
- [ ] 항목 정보에 "draft" 배지 또는 상태 표시
- [ ] 항목 클릭하여 상세 내용 확인 가능

## 3. 통합 검색 기능 테스트

### 3.1 단일 소스 검색
- [ ] 채팅 페이지로 이동
- [ ] "💡 지식베이스"만 체크
- [ ] 질문 입력 및 전송
- [ ] 답변 하단 참고 문서에 지식베이스 항목만 표시됨

### 3.2 다중 소스 검색
- [ ] "📄 문서" + "💡 지식베이스" 체크
- [ ] 질문 입력 및 전송
- [ ] 답변 하단 참고 문서에 문서와 지식베이스 항목이 섞여 표시됨
- [ ] 각 항목에 출처 정보가 명확히 표시됨

### 3.3 웹 검색 포함
- [ ] 모든 소스 체크 (문서 + 지식베이스 + 웹)
- [ ] 질문 입력 및 전송
- [ ] 답변 생성됨
- [ ] 참고 문서 확인 (웹 검색 결과가 없을 수 있음 - 정상)

## 4. UI/UX 일반 테스트

### 4.1 반응형 디자인
- [ ] 브라우저 창 크기 조절
- [ ] 레이아웃이 반응형으로 조정됨
- [ ] 모바일 뷰에서도 사용 가능

### 4.2 다크 모드
- [ ] 시스템 테마 변경 (라이트 ↔ 다크)
- [ ] 페이지 테마가 자동으로 변경됨
- [ ] 모든 텍스트가 읽기 쉬운 대비를 유지함

### 4.3 로딩 상태
- [ ] 데이터 로딩 중 스피너 또는 로딩 메시지 표시
- [ ] 버튼 비활성화 (중복 클릭 방지)
- [ ] 로딩 완료 후 UI 정상 작동

### 4.4 에러 처리
- [ ] 네트워크 에러 발생 시 에러 메시지 표시
- [ ] 유효하지 않은 입력에 대한 validation 메시지
- [ ] 에러 후 정상 작동으로 복구 가능
